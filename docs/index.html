Features

日報を書くだけで、あとから使える形に整う。

01

チェックボックス起点

today を並べて、終わった項目だけにチェックを入れます。完了済みだけが done に変換されるので、 TODO とログが自然に分離されます。

02

impact を残せる

各タスクに impact を添えられるため、半年後に「何をしたか」ではなく 「どんな価値を出したか」で振り返れます。

03

Markdown と GitHub へ蓄積

日報は `daily/YYYY/YYYY-MM-DD.md` に保存。UI から commit / push まで実行できるので、 リポジトリがそのまま自分の仕事ログになります。

Flow

日報から実績までの流れ

1

today を書く

その日やることをアプリに追加する。

2

終わったらチェック

完了したらチェックを入れ、必要なら impact を追記する。

3

Markdown 化

保存時にチェック済みだけが done に正規化される。

4

GitHub に蓄積

UI からそのまま commit / push して、自分のログ資産にする。

Structure

保存される形式

編集中の状態

.work-log-state/entries/YYYY/YYYY-MM-DD.json

正式な日報

daily/YYYY/YYYY-MM-DD.md
