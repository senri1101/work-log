WORK LOG は、Notion に近いブロック編集体験で毎日のログを書き、 その内容を `daily/YYYY/YYYY-MM-DD.md` にそのまま保存する Tauri アプリです。

入手方法を見る GitHubを見る

いまは macOS 向け `.app` の利用を前提にしています。

日報の実体は Markdown なので、Git でそのまま管理できます。

  • 見た目はブロックエディタ、保存形式は Markdown
  • 未着手 / 着手中 / 完了 をそのまま保存
  • UI から commit / push まで完結

Download

今の入手方法

方法 1

ソースから `.app` を作る

現時点ではこの方法が確実です。リポジトリを clone して `pnpm tauri build --bundles app` を実行すると、macOS 用の `work-log.app` を作れます。 

git clone https://github.com/senri1101/work-log.git
cd work-log
pnpm install
pnpm tauri build --bundles app

出力先

生成されるファイル

ビルド後の `.app` は次の場所に出力されます。

src-tauri/target/release/bundle/macos/work-log.app

`.dmg` は環境によって失敗することがあるため、まずは `.app` 配布を前提にしています。

Features

日報を書くだけで、Markdown がそのまま積み上がる。

01

ブロック編集で書ける

BlockNote ベースの UI で、記法を前面に出さずに見出しやチェックリストを編集できます。

02

フルログで残る

完了だけでなく、未着手や着手中、メモ、保留も含めてその日の状態全体を Markdown に保存します。

03

Markdown と GitHub へ蓄積

日報は `daily/YYYY/YYYY-MM-DD.md` に保存。UI から commit / push まで実行できるので、 リポジトリがそのまま自分の仕事ログになります。

Flow

毎日のログを残す流れ

1

ブロックで書く

その日の見出し、チェックリスト、メモを 1 画面で編集する。

2

状態を更新する

未着手、着手中、完了をその場で更新する。

3

Markdown に保存する

保存時にその日の内容全体が `daily/*.md` に反映される。

4

GitHub に蓄積

UI からそのまま commit / push して、自分のログ資産にする。

Structure

保存される形式

編集中の状態

.work-log-state/entries/YYYY/YYYY-MM-DD.json

正式な日報 / バックアップ

daily/YYYY/YYYY-MM-DD.md
